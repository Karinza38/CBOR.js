Intoduction

This document describes a JavaScript API for encoding and decoding CBOR [RFC8949]. The API loosely mimics the "JSON" object by exposing a single global object, unsurprisingly named "CBOR". The core is based on wrapping CBOR data items in type-specific objects. These objects are used for encoding CBOR data, as well as being the result of CBOR decoding. The API provides some specific features including:
  1. Deterministic Encoding
  2. Diagnostic Notation encoder and decoder
  3. Strict type checking of arguments
  4. Optional check for unread elements

JavaScript Number Considerations

Since JavaScript natively only supports Number (backed by 64-bit IEEE 754 floating point data), and BigInt (offering unlimited integers), it is important to understand how these number types can be mapped to CBOR which offers two variants of integers and three variants of floating point types.
Integer Numbers
Due to the Number.MAX_SAFE_INTEGER limitation, CBOR documents using integer elements with a magnitude in excess of 53 bits, must for such elements turn to BigInt, otherwise run-time errors may occurr when accessing integer data using ${getInt()} or during creation of integers using ${CBOR.Int(Number)}. That is, for creation of such integers, ${CBOR.BigInt(BigInt)} is the proper solution, while access requires ${getBigInt()}.
Floating Point Numbers
Although CBOR encoding depends on 16, 32, and 64-bit IEEE 754 variants, the CBOR.js implementation makes this transparent; the API only exposes the Number type.

Diagnostic Notation

Through the ${CBOR.decodeDiagnosticNotation(String)} method, CBOR data may be provided in diagnostic (textual) notation (as described in section 8 of RFC 8949), making CBOR useful for "config" and test data files as well.

By adhering to the Deterministic Encoding scheme above, CBOR data can be bidirectionally converted between its native (binary) format and diagnostic notation without getting corrupted. Note though that text-binary-text "roundtrips" do not necessarily return identical text: 0x10 used as diagnostic notation input will return 16 as diagnostic notation output.

Creating CBOR data in diagnostic notation is provided by ${toString()}.
